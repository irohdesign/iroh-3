Overview

String Theory is a single-page web application that I built while refurbishing my Ibanez 12-string guitar. I wasn’t experienced with vintage instruments, having only worked with newer models, so I went to several stores and looked online for advice on how to properly restring it.

My first job was selling and repairing instruments, so I felt fairly confident going into the process. As I dug deeper, I became more and more frustrated. Store assistance wasn’t very useful because I was often recommended brands with little explanation that didn’t fit my preferences, while blogs and websites gave me text-heavy posts where I needed to dig through 15 paragraphs to find the one applicable section.

The Challenge & The Solution

The right set of strings are vital when playing guitar, but finding that set is difficult. There are hundreds of options and it’s easy to get overwhelmed. Strings are also hard to pick because music is subjective and each musician is unique. They prefer different tones and may just not like how a certain brand feels on the fingertips.

I built the String Theory application because I wanted to streamline the selection process. It’s a simple but effective tool that takes in the user’s experience level and genre choice, and returns with not only string recommendations, but also the reasoning behind those options so users can make informed decisions about their instrument care and sound.

My main frustration while purchasing strings myself is that personal preference isn’t taken into account, so I included the mechanics behind string recommendations to enable users to make the decision for themselves and take the process into their own hands.

Design Process

The biggest hurtle in the design process was establishing effective parameters. I combed through dozens of reviews and articles online to decide on the major determining factors: experience level & genre.


I chose experience level primarily because of my years playing and selling guitars. Beginners lack the necessary fingertip callouses and therefore struggle with thicker strings. Their hands get sore faster so they aren’t able to practice as often and this slows their learning process. I’ve known many people who want to play guitar, but struggle with metal strings and eventually quit.

Research Notes

I chose genre as the second input because string material greatly impacts sound quality and tone. If a head banging metal guitarist picks up a pair of light gauge country strings, they’ll likely break them within the week (and even if they don’t, they would sound too quiet). If a rocker, who’s hoping to solo like Chuck Berry, picks up flat wound jazz strings, he’ll never get the speed he needs.

Most musicians find their preferred set of strings through years of trial and error, which can be very expensive and time-consuming. String Theory personalizes the shopping process by taking the user’s experience level and genre choice into account and focusing on personal preferences.

Build Process

String Theory was designed to be quick and fast, and the code mirrors that quality. The designs in Sketch were clean and modern. I wrote with Bootstrap and jQuery to minimize the CSS and Javascript. The logo is an SVG that is optimized for faster load time.

I started off with HTML text entry fields, using Javascript to convert those strings into lowercase and match them up with a list of expected results. While this worked well when I personally used it, it didn’t stand up to user testing, as users rarely do what you want them to do.

Experience level worked as a text input at first glance because users would need to enter a specific number. It was effective as long as users wrote “1, 2, 3…” but if users entered in “one, two, three” the form would fail.

I changed the experience input to a number stepper to give no room for error and simplify the entry process.

Final version of form

I passed the form to a few friends for experimentation, and they immediately abandoned the recommended inputs and started entering their own sub-genres of music (Electroswing, Nordic Metal, Smooth Jazz). Text entry wasn’t going to work for genres because there are too many options that could be entered. Throwing an error every time a user steps out of line seemed impolite, so I changed the method.

I switched the genre input to a dropdown menu with four options: Jazz, Rock, Blues and Metal. These four genres have the most distinct string requirements and are widely recognized.

The results are built into the page even when not active, because I used CSS to set the display attribute to none. This means that the form does not need to reference another document to retrieve results, and the page doesn’t refresh. This strategy makes the form much faster.

Form results with recommendation and explanation

I was annoyed by the many string recommendations online with no clear link to the products, so I made sure to include three options for each result that have anchors not only in the string’s title, but also in the image. Bigger targets are easier for users to click and make the experience better.

Outcome

String Theory is a personal project that I started largely out of frustration. I wanted a better vehicle for the information, so I built one. I sat down and researched not only the top recommended brands and models, but also the logic behind those choices. I made an interactive tool that simplifies the choice and prioritizes the musician’s experience.

Building this application was good experience because it made me practice good judgement and critical thinking when applying my coding skills. Programming languages and tools are very important, but the ability to think and plan implementation of those technologies is more important when trying to get the job done.
